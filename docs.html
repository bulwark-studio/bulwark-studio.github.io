Documentation

Everything you need to install, configure, and use Bulwark.

Installation

One-Line Install (Linux)

curl -fsSL https://bulwark.studio/install.sh | bash

Detects your system, installs Node.js if needed, clones the repo, runs npm install, and starts Bulwark on port 3001.

npm

git clone https://github.com/bulwark-studio/bulwark.git
cd bulwark
npm install
MONITOR_USER=admin MONITOR_PASS=changeme npm start

Docker

docker build -t bulwark .
docker run -d -p 3001:3001 \
  -e MONITOR_USER=admin \
  -e MONITOR_PASS=changeme \
  -v /var/run/docker.sock:/var/run/docker.sock \
  bulwark

Note: Mount the Docker socket (/var/run/docker.sock) to enable Docker management features.

Create a .env file in the project root or set environment variables directly.

First Login

  1. Open http://your-server:3001
  2. Log in with the MONITOR_USER / MONITOR_PASS credentials
  3. Go to Settings and change your password immediately
  4. Optionally enable TOTP 2FA in Settings > Security

Terminal

Full xterm.js + node-pty terminal in the browser. Supports 256-color, resize, copy/paste, and session persistence.

  • Shell tab — Standard terminal session
  • AI tab — Claude-powered natural language commands
  • Vault tab — Secure credential access

Docker Management

27 native Docker Engine API endpoints. No Portainer needed.

  • Container lifecycle: create, start, stop, restart, remove
  • Real-time stats (CPU, memory, network I/O)
  • Log streaming with search and filtering
  • Image and volume management
  • Connection wizard for remote Docker hosts
  • AI fleet diagnostics

Connection: Bulwark connects to Docker via /var/run/docker.sock (Linux) or TCP. Use the connection wizard in Settings if Docker is on a remote host.

Database Studio

Supabase-style database management with 30+ endpoints. Direct PostgreSQL introspection via information_schema and pg_catalog.

SQL Editor

CodeMirror 5 with syntax highlighting, autocomplete, query history, saved queries, and AI SQL generation.

  • Ask Claude to generate SQL from natural language
  • DDL blocked by default (safety layer)
  • All queries logged to query history

Table Browser

Two-panel explorer: table list on the left, tabs on the right (Columns, Data, Constraints, Foreign Keys, Indexes).

Schema Explorer

Browse functions, triggers, extensions, and indexes.

Migrations

Applied vs. pending migration tracking. Docker test-run (spin up PG, apply, validate, destroy). Schema diff.

Roles

PostgreSQL role manager with AI security audit (scores permissions), permission heatmap, and AI role generator.

Backups

pg_dump / pg_restore integration. AI backup strategy analysis with disaster recovery planning.

Git + Deploy

  • Git view: Commit, push, branch management, diff, log, AI PR descriptions
  • Deploy view: One-click deploy pipeline with pre-flight checks and rollback
  • Git Projects: Manage multiple repositories with dynamic cwd switching

Monitoring

  • Dashboard: Real-time CPU, memory, disk metrics via Socket.IO (3s refresh)
  • Metrics: Historical charts with Chart.js, process list, AI performance analysis
  • Uptime: HTTP/TCP health checks with configurable intervals and alerting
  • Multi-Server: Aggregate metrics across multiple servers

Security Center

  • Security posture scoring
  • Secret scanning and dependency audit
  • SSL/TLS certificate management
  • AES-256-GCM encrypted credential vault
  • AI-generated fix recommendations
  • Cloudflare DNS and tunnel management

BYOK AI Setup

Bulwark uses Bring Your Own Key — you install AI CLI tools on your server and authenticate with your own subscriptions.

ProviderInstallRequirement
Claude CLInpm i -g @anthropic-ai/claude-cliAnthropic subscription
Codex CLInpm i -g @openai/codexOpenAI API key
None—AI features disabled

Configure in Settings > AI Provider. Bulwark auto-detects installed CLIs.

Privacy: AI requests go directly from your server to your AI provider. Bulwark never sees your prompts, responses, or API keys.

API Endpoints

270+ REST API endpoints across 31 route modules. All endpoints require authentication via session cookie or Bearer token.

# Example: Get system info
curl -b "monitor_session=TOKEN" http://localhost:3001/api/system

# Example: List database tables
curl -b "monitor_session=TOKEN" http://localhost:3001/api/db/tables

# Example: Execute SQL query
curl -X POST -b "monitor_session=TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"sql":"SELECT NOW()"}' \
  http://localhost:3001/api/db/query

Dimension Dark Theme

Glass-morphism dark theme with a strict two-color signal system:

  • Cyan (#22d3ee) — success, healthy, active, positive
  • Orange (#ff6b2b) — error, warning, destructive, down

Glass effects: backdrop-filter: blur(20px) saturate(180%) on all cards, sidebar, topbar, and modals. JetBrains Mono typography throughout.

Troubleshooting

Server won't start

  • Ensure Node.js 18+ is installed: node --version
  • Check port 3001 is available: lsof -i :3001
  • Check logs for errors at startup

Database features unavailable

  • Set DATABASE_URL in your .env file
  • Ensure PostgreSQL is running and accessible
  • The app works without a database in degraded mode

Docker not connecting

  • Ensure Docker is installed and running
  • Check socket permissions: ls -la /var/run/docker.sock
  • For Docker Desktop, use the connection wizard in Settings

AI features not working

  • Install Claude CLI: npm i -g @anthropic-ai/claude-cli && claude auth
  • Verify it works: claude --print "hello"
  • Check Settings > AI Provider is set correctly
